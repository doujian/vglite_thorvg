### Key Implementation Notes

1. **ThorVG Architecture**: ThorVG's GlCanvas manages GPU resources internally and handles CPU-GPU synchronization during `canvas->sync()` (called in `vg_lite_finish()`).

2. **No Direct GL Calls**: ThorVG's minimal GL interface doesn't expose `glMemoryBarrier()` or `glFlushMappedBufferRange()`. The implementation returns `VG_LITE_SUCCESS` for both backends since ThorVG handles synchronization internally.

3. **Backend-Specific Logic**:
   - GL Backend: Returns `VG_LITE_SUCCESS` - ThorVG handles GPU sync during `sync()`
   - SW Backend: Returns `VG_LITE_SUCCESS` - No GPU sync needed, data is in CPU memory

4. **Parameter Validation**: The function validates that `buffer != nullptr` and returns `VG_LITE_INVALID_ARGUMENT` for null buffers.
